# pre-commit-hooks Specification

## Purpose

Versioned pre-commit hooks that run the full local gate (lint, vet, build, GGA) before every commit. Replaces the unversioned `.git/hooks/pre-commit` (generated by `gga install`) with a shareable `.githooks/pre-commit` activated via `core.hooksPath`.

## Requirements

### REQ-PCH-001: Versioned Pre-Commit Hook

The repository MUST contain a versioned pre-commit hook at `.githooks/pre-commit`. A new `task setup` command MUST configure `core.hooksPath` to `.githooks`. Fresh clones MUST NOT have hooks active until `task setup` is run.

#### Scenario: fresh clone has no hooks

- GIVEN a fresh `git clone` of the repository
- WHEN inspecting `.git/hooks/pre-commit`
- THEN no pre-commit hook exists (hooks are not active by default)

#### Scenario: task setup installs hooks

- GIVEN a developer runs `task setup`
- WHEN the command completes
- THEN `git config core.hooksPath` MUST return `.githooks`
- AND `.githooks/pre-commit` MUST be executable

#### Scenario: hook runs full local gate

- GIVEN `.githooks/pre-commit` is active
- WHEN a commit is attempted
- THEN the hook MUST run `golangci-lint run`, `go vet ./...`, `go build ./...`, and `gga run` in sequence

---

### REQ-PCH-002: Pre-Commit Runs Deterministic Checks

The pre-commit hook MUST run deterministic checks (golangci-lint, go vet, go build) before allowing a commit. Any failure MUST block the commit.

#### Scenario: lint failure blocks commit

- GIVEN `golangci-lint run` would report violations
- WHEN a commit is attempted
- THEN the commit is blocked (non-zero exit)

#### Scenario: vet failure blocks commit

- GIVEN `go vet ./...` would report issues
- WHEN a commit is attempted
- THEN the commit is blocked (non-zero exit)

#### Scenario: build failure blocks commit

- GIVEN `go build ./...` would fail
- WHEN a commit is attempted
- THEN the commit is blocked (non-zero exit)

#### Scenario: all checks pass allows commit

- GIVEN all deterministic checks pass
- WHEN a commit is attempted
- THEN the commit proceeds to GGA check

---

### REQ-PCH-003: GGA in Pre-Commit

The pre-commit hook MUST run `gga run` after deterministic checks pass. The `--no-verify` escape hatch MUST be supported per AGENTS.md rule #41 (requires `NO-VERIFY: <reason>` in commit body + follow-up fix commit).

#### Scenario: GGA passes allows commit

- GIVEN all deterministic checks pass and `gga run` exits 0
- WHEN a commit is attempted
- THEN the commit is created

#### Scenario: GGA fails blocks commit

- GIVEN all deterministic checks pass but `gga run` exits non-zero
- WHEN a commit is attempted
- THEN the commit is blocked (non-zero exit)

#### Scenario: --no-verify with NO-VERIFY reason allows commit

- GIVEN a developer uses `git commit --no-verify`
- WHEN the commit includes `NO-VERIFY: <reason>` in the commit body
- THEN the commit is created (bypassing all hook checks)
- AND a follow-up fix commit is expected per AGENTS.md
